jamming_by_growth.f

Behavior and outputs

  • Adds a new STEPLOG_* file with a header and per-step records for step index, particle count, energy per particle, pressure, time step, packing fraction, total growth rate, and jamming-state flags.
  • Computes phi and total_growthrate every iteration before writing the step log.
  • Replaces direct writes of rate(i) * exp(-PP(i)/P0) with a guarded ratei_eff value so the output does not evaluate that expression when P0 <= 0 or PP(i) <= 0.

Geometry and internal cleanup

  • Introduces mol_to_atoms to centralize the conversion from cell-level state to atom-level positions and effective diameters.
  • Updates makelist to use the same aspect-ratio-dependent upper-bound diameter logic as the new helper, instead of the previous one-branch approximation.
  • Removes unused state and arguments including D0, copied D0 arrays, unused bud counters, and unused common-block values PT and PR.
  • Adds more explicit IMPLICIT NONE usage and explicit local declarations in several Numerical Recipes-style routines.

shear_yeast_linearshear.f

Behavior and outputs

  • The shear statistics file now starts with a header and records additional state: phi, Nmm, Nbb, and Nmb for each strain step.
  • Reads the configuration header directly into phi and computes the per-particle rotational scale once before the shear loop, rather than rebuilding it every step.
  • Changes the shear-stress contribution formula to include a geometric weighting factor dij**2 / di1j1**2 in the pair-force accumulation.

Geometry and contact logic

  • Adds the shared mol_to_atoms helper and reuses it in the energy, force, and contact routines to keep geometry construction consistent.
  • Updates neighbor-list generation to use the same aspect-ratio-dependent diameter upper bound as the geometry helper.
  • Simplifies contacts_yeast by removing unused parameters and deriving the atom-level geometry through the shared helper instead of duplicating the conversion logic.
  • Adds the in-source note that CG_check is intentionally conservative and should not be treated as a bug in future audits.

Notes For PR Use
