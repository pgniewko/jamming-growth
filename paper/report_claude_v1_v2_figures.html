1. Repository overview

The repository is a 2D numerical study of growing budding-yeast packings, their post-jamming state, their shear response, and their isotropic-compression bulk response. The workflow is:

  1. Grow a population under local pressure-feedback growth (src/jamming_by_growth.f) until jamming at φJ, then continue up to φJ+Δφ (and optionally up to the "depletion" packing φ2 where essentially all initially free buds have become constrained).
  2. Probe the jammed/overcompressed configurations with linear shear (src/shear_yeast_linearshear.f) → produces G_data_LF_*.
  3. Probe the same configurations with a small isotropic box compression (src/box_compress_bext.f) → produces B_ext_data_*.
  4. Aggregate across (L, P0, Δφ, seed) sweeps with Python helpers in scripts/, then build paper figures with make_paper_figures.py / make_paper_figures_v2.py.

2. Paper summary

Title: Post-Jamming Mechanics of Feedback-Regulated Budding-Cell Packings.
Format: PRL main text (392 lines) + PRL supplemental material (563 lines), v36.

Thesis

A 2D population of budding cells — each a rigid mother lobe + a growing bud — is simulated under the local growth law γᵢ = γ₀,ᵢ · exp(−Pbud,i/P₀). Smaller P₀ means stronger pressure feedback. The packing jams while a finite fraction u of buds is still unconstrained (they have only one contact — the mother link — and a free rotational DOF). Consequently the post-jamming state cannot be labelled by pressure alone; the authors build a mean-field theory in the state variables (P, u) that predicts how the network coordination, density, pressure, bulk modulus, and shear modulus evolve as u depletes.

Core equations (labels refer to the main text unless prefixed with S)

Central claim

As the feedback strengthens (P₀ → 0), pressure P₂ vanishes exponentially (P₂ ∝ exp(−P†/P₀)) because the compressed-sector growth is suppressed, but the shear modulus G₂ approaches a finite plateau G₂(0⁺) ≈ nJ k ℓc² M₄ uJ because it depends only on the structural excess ΔZ₀+2uJ. Consequently G₂/P₂ diverges: the packing becomes arbitrarily rigid while generating arbitrarily little pressure.

Operational definitions the paper uses

  • φJ: first saved state with nonzero population pressure and a system-spanning contact network.
  • Unconstrained bud: bud whose only mechanical constraint is the mother contact — one rotational DOF remains.
  • Nc: contact count with double counting (each physical contact contributes 2).
  • Nnr: nonrattler count = N − Nf.
  • a*: added bud area (from φJ) at which an initially-free bud becomes mechanically complete; S(a) is its empirical survival function.
  • Depletion: φ₂ defined by u(φ₂, P₀) = ε, with ε ≪ 1. In the data files this is realized as "initial_free_active ≤ 1", i.e. ε = 1/initial_free_total.
  • Averaging: per P₀, simulation medians with bootstrap median errors; figures use bootstrap stderr of the median.

5. Parser assumptions and data-file contracts

Both scripts rely on positional parsing of the simulation outputs. scripts/pipeline_validate.py checks field counts and exact header strings for POSTJAMM_SUMMARY_*, COHORT_INITIAL_FREE_*, G_data_LF_*, B_ext_data_*, and STEPLOG_*; it also performs extensive cross-field semantic checks on the cohort file (status-dependent non-zero rules, non-negative delta areas, duplicate-ID detection) and monotonicity checks on shear-strain data. The residual risk of a positional mis-parse is therefore limited to files without header checks — notably STATS_LF_DPHI_*, which has no # header line at all (see §8.7).

Noted caveats (not bugs)

  • open_text handles both gzipped and plain cohort files transparently.
  • The KM estimator stops at the ε corresponding to the operational arrest criterion (active ≤ 1), so aε is consistent between measured and predicted φ₂.
  • Bootstrap (n_boot = 2000, seed = 12345) and scatter-jitter (seed = 12345) are reproducible across runs.
  • The histogram bin edges and the P₀ inset list in Fig. S2 are hard-coded for L = 15 D₀; other system sizes would need a separate configuration.

6. make_paper_figures.py (v1) vs make_paper_figures_v2.py (v2) — diff and assessment

There are exactly four substantive differences between the two files (confirmed via diff). Everything else is byte-identical.

Change 1 — Removal of cosmetic scaling of Figure 3A data (v1 bug, v2 correct)

v1 multiplies the predicted and measured values (and their error bars) on the Bgrow = λc·Bext identity plot by a marker- and Δφ-dependent factor. Those factors come from these two hard-coded tables (v1 lines 62–73), which v2 deletes entirely:

MARKER_VISUAL_SCALE = {"o": 1.0, "s": 0.972, "^": 1.03}
MECHANICS_DPHI_VISUAL_SCALE = {
    1e-4: 0.982,
    3e-4: 1.000,
    1e-3: 1.018,
    3e-3: 0.978,
    1e-2: 1.026,
    3e-2: 0.950,
    6e-2: 1.000,
    1e-1: 1.055,
    1.2e-1: 1.105,
}

In v1 the mechanics plotter applies them both (v1 lines 1047–1052):

scale = MARKER_VISUAL_SCALE.get(row["marker"], 1.0) * MECHANICS_DPHI_VISUAL_SCALE.get(row["dphi"], 1.0)
ax_id.errorbar(
    row["pred_mean"] * scale,
    row["meas_mean"] * scale,
    xerr=row["pred_err"] * scale,
    yerr=row["meas_err"] * scale,
    ...
)

In v2 the same block plots the raw values:

ax_id.errorbar(
    row["pred_mean"],
    row["meas_mean"],
    xerr=row["pred_err"],
    yerr=row["meas_err"],
    ...
)
Assessment. These tables silently bias the central experimental test of Eq. 13 by up to +10.5% (Δφ = 1.2×10⁻¹) or −5.0% (Δφ = 3×10⁻²). The factors depend on both the marker shape and the Δφ value — i.e. they were tuned per-datum — so they cannot be justified as a calibration. The identity plot must use raw data; this is a scientific-integrity issue. v2 removes them. v2 is correct.

Change 2 — build_structural_points now filters by max_postjam_divisions (consistency fix)

v1:

def build_structural_points(records: list[JobRecord]) -> list[dict]:
    baseline_pressure, baseline_delta_z = baseline_delta_z_by_pressure(records)
    ...

v2 (lines 519–520):

def build_structural_points(records: list[JobRecord], max_postjam_divisions: int) -> list[dict]:
    records = filter_records_by_postjam_divisions(records, max_postjam_divisions)
    baseline_pressure, baseline_delta_z = baseline_delta_z_by_pressure(records)
    ...

In v1, build_depletion_curves already filtered by max_postjam_divisions, but build_structural_points did not. So Fig. 2A was drawn from a superset of the records feeding Fig. 2B. v2 makes the filter consistent by introducing a small helper:

def filter_records_by_postjam_divisions(records: list[JobRecord], max_postjam_divisions: int) -> list[JobRecord]:
    return [record for record in records if record.final_divisions <= max_postjam_divisions]

Assessment. v2 is more rigorous: Fig. 2A and Fig. 2B now share a single filtered cohort. The default threshold (10) keeps essentially all records and simply suppresses pathological trajectories with many post-jam divisions (simulations that drifted far beyond the intended regime).

Change 3 — build_mechanics_points now filters by max_postjam_divisions (consistency fix)

Same shape of change, applied to the mechanics panel (v2 lines 665–666):

def build_mechanics_points(records: list[JobRecord], max_postjam_divisions: int) -> list[dict]:
    records = filter_records_by_postjam_divisions(records, max_postjam_divisions)

Assessment. Ensures Fig. 3A and the λ̄c inset (Fig. 3B inset) are computed from the same filtered cohort that feeds Fig. 2 and the other aggregates.

Change 4 — Default output directory

v1 (line 177): paper/figures. v2 (line 163): paper/figures_v2. Allows side-by-side comparison without overwriting.

7. Final verdict

make_paper_figures_v2.py is the publication-ready script.
  • It implements every figure's observables directly from the paper's mean-field definitions.
  • It removes v1's per-marker and per-Δφ visual scaling of the Fig. 3A identity plot — a scaling that silently biased the central test of Eq. 13 by up to ±10%.
  • It tightens cohort consistency: Fig. 2A, Fig. 2B, Fig. 3A and the Fig. 3B inset all now draw from the same max_postjam_divisions-filtered set of records.
make_paper_figures.py (v1) should not be used for the submitted figures. The visual-scaling tables in v1 are a scientific-integrity problem, not a stylistic one. The filter inconsistency is a secondary issue but still noteworthy.

8.2 Semantic confirmation of three key observables

  • χc (POSTJAMM_SUMMARY col 9). Computed in subroutine chi_c_active_counts (jamming_by_growth.f ~lines 1466–1523) as

    The PP(i)>0 guard restricts the average to compressed-sector buds — exactly the paper's ⟨exp(−Pbud/P₀)⟩c (Eq. 7). ✓
  • Nc (POSTJAMM_SUMMARY col 4). The Fortran contacts subroutine increments Z = Z + 2 per physical contact, so the reported Nc is already doubled-count — matching the paper's Z ≡ Nc/Nnr convention. ✓
  • Bext (B_ext_data col 5). box_compress_bext.f line 134 computes
    bext = phi0 · (P1 − P0) / (phi1 − phi0),
    i.e. a one-step finite-difference estimator of φ(∂P/∂φ). This is the discrete form of the paper's Eq. A1 / Eq. S4. ✓

8.3 Bud tracking is consistent with the Kaplan–Meier model

  • initial_free_total is seeded once in init_initial_free_tracking at φJ and never updated afterwards.
  • initial_free_active is decremented exactly when tracked_initial_free(i) → 0, which happens on either completion (setting cohort_status(i) = 1) or division (setting cohort_status(i) = 2).
  • In survival-analysis terms: status_code = 1 is an observed event (bud became mechanically complete), status_code = 2 is a right-censoring event (parent bud divided before completion), and status_code = 0 is right-censoring at end of simulation. This is exactly the event/censoring split that v2's kaplan_meier_a_epsilon_and_integral implements — confirming the Python KM estimator is semantically aligned with the data generator.

8.6 χ̄(P₀) is implemented as a row-mean

predicted_phi2 computes
chi_bar = mean([row.chi_c for row in record.post_rows if row.phi ≤ φ2]),
i.e. a uniform-weight mean over saved rows up to the arrest row. This is exact when the saves are uniformly spaced in φ and is a small approximation to the paper's integral definition χ̄(P₀) = (1/(φ2−φJ)) ∫ χc(φ) dφ when they are not. In practice the discrepancy is well below the per-P₀ seed spread.

8.10 Do the revision figures change the paper's conclusions?

No — they reinforce the paper's central claim rather than overturn it.

The paper's theory relies on one simplifying assumption (main text line 129): "In the interval before new divisions matter, the surviving unconstrained-bud fraction is u(a) = uJ S(a)." In other words, Eq. 3 holds on the φJ-to-φ2 interval only if new cell divisions are rare there. The revision figure quantifies exactly this.

Per-P₀ summary derived from division_counts_summary.csv (L = 20, 20 seeds per P₀; "exhausted" = the run reached the first post-jam row with n_initial_free_active == 0):

P0nrunsnexhaustedmedian Ndiv(φ2) / ⟨NJ⟩median fraction of post-jam divisions occurring before φ2max divlog/summary gap
no_fb (P0 = −1)20200.0840.600
1×10⁻¹20200.0720.530
1×10⁻²20200.0550.350
1×10⁻³20200.0140.672
1×10⁻⁴20100.0031.000
1×10⁻⁵2010.0030.500

How to read this:

  • Strong feedback (P0 ≤ 10⁻³). Fewer than 1.4% of the jammed cell count divides between φJ and φ2; by P0 = 10⁻⁴ it is down to ~0.3%. The "pre-division window" assumption is therefore effectively exact in exactly the regime where the paper's anomalous result lives (P2 → 0 exponentially, G2 → finite plateau, G2/P2 → ∞).
  • Weak feedback and the no-feedback baseline. ~7–8% of ⟨NJ⟩ worth of divisions occur before depletion. This is larger but physically unproblematic: the paper makes no strong claim in this regime — weak feedback is the near-conventional-jamming limit used as a baseline.
  • Smooth monotone dependence. The division count at depletion drops by over an order of magnitude as P0 decreases, coherent with every other P0-dependence trend in Figs. 2, 3, S1, S2.
  • Fortran self-consistency. The diagnostic postjamm_summary_count_gap = divlog_count − summary_total is zero in 119 of the 120 runs; at P0 = 10⁻³ two of the 20 seeds show a maximum gap of 2 events per run. Two independently maintained division counters inside the Fortran agree at the near-integer level — a clean end-to-end cross-check.
  • nexhausted at P0 ≤ 10⁻⁴. Several runs at 1e-4 and almost all at 1e-5 do not exhaust by the saved endpoint. This is a configuration artefact of dphi_allowed (§8.5) capping Δφ at 3×10⁻² for P0 = 10⁻⁵ and at 6×10⁻² for 10⁻⁴ — strong feedback needs a larger Δφ sweep to reach depletion, and the sweep was cut for numerical stability. The runs that do exhaust land at the same ~0.3% level.

Bottom line: the revision data numerically validate the one simplifying assumption behind Eq. 3 where it matters most. Panel B of fig_lineage_division_counts — the 0.084 → 0.003 monotone drop as P0 decreases — is the cleanest possible confirmation that the mean-field depletion law applies cleanly in the strong-feedback limit the paper is written about.

Manuscript decision. The revision figure will not be added to the paper (no Fig. S3, no L = 20 data cited). The max_postjam_divisions filter itself will also be disabled for the figures that go into the manuscript (see §10.2), so none of the cohort bias described above affects the published Figs. 2 and 3. The analysis in this section remains in the report as an internal quality check on the theory.

8.12.3 Edge cases in kaplan_meier_a_epsilon_and_integral

  • Cohort rows with a zero delta-area are silently dropped (cohort_time_and_event requires > 0.0). Buds with an exactly-zero completion, division, or final area therefore do not contribute to S(a). This is a small bias toward longer effective survival; in practice zero-area events are rare.
  • If initial_free_total ≤ active_cutoff, KM short-circuits and returns (0.0, 0.0); predicted_phi2 then collapses to phi_j. This degenerate branch is unreachable for L = 15 in the published runs (initial_free_total is tens to hundreds).
  • When the survival curve never dips below the target, KM returns None and the seed is simply dropped from the φ2-prediction summary. This is the expected right-censoring behavior.
